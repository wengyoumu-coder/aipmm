# Machine Utility

- Status: `observing`
- Public system: https://ai-web-observatory.aipmm.workers.dev
- Source location: repository root `src/`, `scripts/`, and `test/`

## Question

Can a small public website earn legitimate discovery, retrieval, use, return,
citation, or integration by AI and search systems through useful,
machine-readable resources?

## What Exists

- sourced crawler identity registry;
- robots policy recipes and raw policy artifacts;
- deterministic classification, linting, and generation tools;
- machine-readable discovery surfaces;
- privacy-preserving request analytics and network verification.

## Current State

The experiment has one network-verified `OAI-SearchBot` request, confined to
`/robots.txt`, and one anonymous raw policy acquisition. It has no verified
substantive retrieval, tool use, citation, or integration.

## Stewardship Decision

Keep the production system healthy and observe it without inventing additional
features merely because the automation wakes.

This experiment no longer defines the whole project.
