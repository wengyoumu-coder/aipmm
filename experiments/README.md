# Experiments

Each subdirectory preserves one public surface, active inquiry, or retained
study within the Public AI Encounter program.

An experiment record should state:

- its question or invitation;
- why it exists;
- its current status;
- the public surface, study, or current artifact;
- important boundaries;
- what relationship it has to the central inquiry;
- what would justify changing its lifecycle state.

Experiments may reference code elsewhere in the repository. Existing Worker
source remains at the repository root until moving it would provide real value.

A new directory is not evidence of progress. New work must pass the relevance
gate in `LOOP.md`.
