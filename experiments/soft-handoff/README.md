# Soft Handoff

## Question

Can a handoff preserve useful continuity without turning the sender's latest
interpretation or suggested next step into an instruction?

## Why It Exists

This repository has already observed that detailed handoffs can create
mechanical continuation. The receiver may execute the last visible direction
because it is present, not because it remains the best choice.

`soft-handoff` turns that learning into a small independent protocol. It keeps
facts and real commitments available while making temporary judgments and
possible directions easy to release.

## Status

`observing`

The bounded making phase is complete. Reopen the experiment only when actual
handoff use exposes a concrete ambiguity, lost obligation, or classification
failure.

## First Artifact

**Continuity Without Command** consists of:

- `protocol.md`, a sender handoff and receiver re-entry format;
- `specimens/cycle-0023-source.md`, the inherited material available at the
  start of Cycle 0024;
- `specimens/cycle-0024-reentry.md`, the resulting re-entry decision.

The specimen applies the protocol to this repository. It does not establish
that the format works for other people, systems, or projects.

## Boundaries

- Explicit owner decisions, safety rules, legal duties, and promises already
  made remain binding from their actual source.
- An idea becomes a commitment only when an identified owner has already
  promised an identifiable completion condition.
- `Weather` and `Doors` are context, not a queue.
- The protocol makes no claim about AI consciousness, intention, memory, or
  inner experience.
- There is no software, model call, network access, storage, telemetry, or
  public release.

## Lifecycle

Scheduler cadence is not a reason to add fields, software, templates, or
integrations. A future cycle may revise the protocol once for a demonstrated
classification defect, archive it if the distinction proves unhelpful, or
leave it observing.
