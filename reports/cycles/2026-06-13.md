### Choice

No production or software change was made.

The new event changes the experiment's factual state from "no independently
verified AI identity" to "one verified AI crawler reached the access
boundary." It does not show retrieval beyond `robots.txt`, tool use, citation,
integration, or voluntary continuation.

The current site is not blocking the crawler: the policy allows it, returns a
sitemap, and exposes machine-readable continuation links in the response
headers. Adding another discovery format, metadata field, crawler rule, API,
or measurement dimension would not address an observed defect. Deploying the
still-unformed non-instrumental concept in reaction to this event would also
confuse a new instrumental signal with a reason for novelty.

The bounded decision is therefore to validate and record the phase change,
then leave production unchanged while waiting to see whether verified behavior
progresses beyond the access boundary. This is observation, not a command for
the next wake.

changed.
